Nostr OAuth Signer

Bridge traditional OAuth authentication with Nostr's cryptographic identity model

What It Does

Nostr OAuth Signer lets users authenticate with familiar OAuth providers — Google, GitHub, Microsoft, or Apple — and then manage Nostr identities (keypairs) within an encrypted vault. Nostr clients connect to the signer via NIP-46 (Nostr Connect) for remote signing, so private keys never need to be exposed to client applications.

Key Features

  • OAuth Authentication — Sign in with Google, GitHub, Microsoft, or Apple.
  • Encrypted Key Storage — NSECs encrypted at rest with AES-256-GCM, derived keys via HKDF.
  • NIP-46 Bunker Protocol — Full Nostr Connect implementation for remote signing.
  • Multi-Identity Support — Manage multiple Nostr keypairs per user.
  • Time-Limited Assignments — Grant temporary identity access with automatic expiration.
  • Admin Dashboard — Web UI for managing identities, users, and assignments.
  • Split Architecture — Production deployment with NSECs isolated on LAN.

How It Works

  1. User authenticates via an OAuth provider.
  2. User selects a Nostr identity from their assigned pool.
  3. Nostr clients connect via NIP-46 for signing — keys never leave the bunker.
