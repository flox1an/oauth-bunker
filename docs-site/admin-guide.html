Admin Guide

The admin dashboard lets you manage Nostr identities, users, and access assignments. Admin routes are at /api/admin/* and should only be accessible from your LAN.

Accessing the Admin Panel

  • Navigate to http://localhost:3000 (or your LAN IP)
  • The admin page is at the /admin route in the web UI
  • In split architecture, admin endpoints only exist on the bunker (LAN-only)
  • Admin API routes: /api/admin/identities, /api/admin/users, /api/admin/assignments

Managing Identities

Adding an identity

  • Click "Add Identity" in the admin panel
  • Paste a Nostr nsec (bech32 format, starts with nsec1...)
  • Optionally add a label (e.g., "Blog", "Personal", "Work")
  • The nsec is immediately encrypted with AES-256-GCM
  • Encryption uses a key derived from MASTER_KEY via HKDF-SHA256
  • The plaintext nsec is never stored — only the encrypted blob and nonce

Viewing identities

  • Each identity shows: public key (npub format), label, creation date, active connection count
  • The npub can be shared — it's the public identifier for the Nostr identity

Deleting an identity

  • Removes the encrypted nsec from the database
  • Disconnects all active NIP-46 connections using that identity
  • Removes all user assignments for that identity
  • This action is irreversible — make sure you have the nsec backed up elsewhere

Managing Users

  • Users are created automatically when they first sign in via OAuth
  • Each user has: ID, OAuth provider (Google/GitHub/Microsoft/Apple), email, avatar URL, creation date
  • The admin panel shows all registered users
  • Users cannot be created manually — they must authenticate via OAuth

Identity Assignments

Creating an assignment

  • Select a user and an identity to assign
  • Choose a duration: 1 day, 1 week, 1 month, 6 months, or 1 year
  • The user can now select this identity when connecting via NIP-46

How assignments work

  • Users can only see and use identities they're assigned to
  • When a user authenticates via OAuth and reaches the identity picker, only their assigned (non-expired) identities appear
  • Assignment expiration is enforced at signing time — the bunker checks before signing each event
  • A background task runs every 5 minutes to clean up expired assignments

Revoking access

  • Delete the assignment in the admin panel to immediately revoke
  • The user will no longer see the identity in their picker
  • Existing NIP-46 connections using that identity will fail at next signing attempt

Monitoring Connections

  • View active NIP-46 connections per identity
  • Each connection shows: client public key, relay URL, creation date, last used timestamp
  • Connections represent active Nostr client sessions (e.g., Habla, Amethyst)
  • Connections can be viewed at /api/connections

Common Admin Tasks

Onboarding a new user

  1. Have the user sign in via any OAuth provider
  2. Find them in the Users list
  3. Create assignments for the identities they should access
  4. Share the bunker connection string (bunker URI) with them

Rotating an identity

  1. Generate a new nsec externally
  2. Add the new identity in admin
  3. Reassign affected users to the new identity
  4. Delete the old identity (disconnects all sessions)

Emergency revocation

  1. Delete the identity's assignments to block new connections
  2. Or delete the identity entirely to disconnect all sessions immediately
