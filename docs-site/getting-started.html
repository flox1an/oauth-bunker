Quick Start

Prerequisites

  • Rust 1.70+ (for building the server)
  • Node.js 18+ (for building the web UI)
  • Git
  • OpenSSL (for generating the master key)

Clone and Build

git clone https://github.com/ArcadeLabsInc/oauth-signer.git
cd oauth-signer

# Build the web UI
cd web-ui && npm install && npm run build && cd ..

# Build the Rust server
cargo build --release

Generate a Master Key

The master key encrypts all stored Nostr private keys. Generate a random 32-byte key:

openssl rand -hex 32

Keep this key secure and backed up. Losing it means losing access to all stored keys.

Create Your Configuration

Create a .env file in the project root:

HOST=127.0.0.1
PORT=3000
PUBLIC_URL=http://localhost:3000
MASTER_KEY=<paste your 64-character hex key>

# OAuth providers (configure at least one)
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
GITHUB_CLIENT_ID=
GITHUB_CLIENT_SECRET=
MICROSOFT_CLIENT_ID=
MICROSOFT_CLIENT_SECRET=
APPLE_CLIENT_ID=
APPLE_CLIENT_SECRET=

# Nostr relays (defaults work for most setups)
# NOSTR_RELAYS=wss://relay.nsec.app,wss://relay.damus.io,wss://nos.lol

Note: All OAuth provider vars are required even if empty. See OAuth Setup for how to get credentials.

Configure an OAuth Provider

You need at least one OAuth provider configured. The quickest to set up is GitHub:

  1. Go to GitHub Developer Settings → OAuth Apps → New OAuth App
  2. Set callback URL to http://localhost:3000/auth/github/callback
  3. Copy Client ID and Client Secret to your .env

See the full OAuth Setup Guide for all providers.

Run the Server

cargo run

Or use the release build:

./target/release/oauth-signer

Verify It's Working

Visit http://localhost:3000 in your browser. You should see the landing page with OAuth sign-in buttons.
