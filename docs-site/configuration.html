Configuration Reference

All configuration is via environment variables, loaded from a .env file in the project root.

Environment Variables

Variable Required Default Description
HOST No 127.0.0.1 Server bind address
PORT No 3000 Server port
PUBLIC_URL No http://localhost:3000 Full public URL used for OAuth callback generation
MASTER_KEY Yes — 32-byte hex key (64 chars) for encrypting stored nsecs
GOOGLE_CLIENT_ID Yes — Google OAuth client ID
GOOGLE_CLIENT_SECRET Yes — Google OAuth client secret
GITHUB_CLIENT_ID Yes — GitHub OAuth client ID
GITHUB_CLIENT_SECRET Yes — GitHub OAuth client secret
MICROSOFT_CLIENT_ID Yes — Microsoft OAuth client ID
MICROSOFT_CLIENT_SECRET Yes — Microsoft OAuth client secret
APPLE_CLIENT_ID Yes — Apple Services ID (e.g., com.yourdomain.app.web)
APPLE_CLIENT_SECRET Yes — Apple JWT client secret (regenerate every 6 months)
NOSTR_RELAYS No wss://relay.nsec.app,wss://relay.damus.io,wss://nos.lol Comma-separated Nostr relay URLs
DATABASE_URL No oauth-signer.db SQLite database file path
ADMIN_PUBKEYS No — Comma-separated hex pubkeys of admin users (for NIP-98 auth)
RUST_LOG No — Logging filter (e.g., info, oauth_signer=debug)

Server Configuration

  • HOST: Bind address. Use 0.0.0.0 to listen on all interfaces, 127.0.0.1 for localhost only.
  • PORT: TCP port. Default 3000.
  • PUBLIC_URL: The full URL where the app is accessible. Critical for OAuth — callback URLs are generated from this. Must match what's configured in each OAuth provider's dashboard.

Encryption

  • MASTER_KEY: The root encryption key. All nsecs are encrypted using keys derived from this via HKDF-SHA256. Generate with: 
    openssl rand -hex 32
    Must be exactly 64 hex characters (32 bytes). Back this up securely — without it, encrypted nsecs cannot be decrypted.

OAuth Providers

All 8 OAuth variables (4 providers × 2 each) are required at startup. If you don't use a provider, you still need to set the vars (can be empty strings, but the app will error if they're missing entirely). See OAuth Setup for how to obtain credentials.

Nostr

  • NOSTR_RELAYS: The relays the bunker connects to for NIP-46 communication. Clients must use the same relays to reach the bunker. The default set (relay.nsec.app, relay.damus.io, nos.lol) provides good coverage.

Database

  • DATABASE_URL: Path to the SQLite database file. Created automatically on first run. Uses WAL journal mode for concurrent access. Relative paths are relative to the working directory.

Logging

  • RUST_LOG: Controls log verbosity using the tracing-subscriber EnvFilter format. Examples:
    • info — general info messages
    • oauth_signer=debug — debug output from the app
    • oauth_signer=trace,tower_http=debug — very verbose with HTTP request logging

Logs are output as structured JSON (useful for log aggregation tools).

Complete Example .env

# Server
HOST=127.0.0.1
PORT=3000
PUBLIC_URL=https://bunker.yourdomain.com

# Encryption
MASTER_KEY=your-64-character-hex-key-here

# Google OAuth
GOOGLE_CLIENT_ID=123456789.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-xxxxx

# GitHub OAuth
GITHUB_CLIENT_ID=Iv1.xxxxxxxxxxxx
GITHUB_CLIENT_SECRET=xxxxxxxxxxxxxxxxxxxx

# Microsoft OAuth
MICROSOFT_CLIENT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
MICROSOFT_CLIENT_SECRET=xxxxxxxxxxxxxxxxxxxx

# Apple Sign In
APPLE_CLIENT_ID=com.yourdomain.app.web
APPLE_CLIENT_SECRET=eyJhbGciOiJFUzI1NiIs...

# Nostr
NOSTR_RELAYS=wss://relay.nsec.app,wss://relay.damus.io,wss://nos.lol

# Database
DATABASE_URL=oauth-signer.db

# Logging
RUST_LOG=info
