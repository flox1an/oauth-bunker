Troubleshooting

Common issues and how to resolve them.

Startup Errors

"MASTER_KEY must be set"

  • The MASTER_KEY environment variable is missing from your .env file
  • Generate one: 
    openssl rand -hex 32

"MASTER_KEY must be 32 bytes"

  • The key must be exactly 64 hex characters (32 bytes)
  • Check for accidental spaces or line breaks

"MASTER_KEY must be valid hex"

  • The key contains non-hex characters
  • Only use characters 0-9 and a-f

"GOOGLE_CLIENT_ID must be set" (or any OAuth var)

  • All OAuth provider variables are required, even if you don't use that provider
  • Set unused providers to empty strings or placeholder values

"Failed to bind to address"

  • The port is already in use: check with lsof -i :3000
  • Or the HOST address is invalid

OAuth Errors

"Redirect URI mismatch" / "redirect_uri_mismatch"

  • The PUBLIC_URL in your .env doesn't match what's configured in the OAuth provider's dashboard
  • Check that the callback URL is exactly {PUBLIC_URL}/auth/{provider}/callback
  • Common issue: using http:// in the provider but https:// in PUBLIC_URL (or vice versa)

"Invalid client" / "unauthorized_client"

  • Wrong client ID or client secret
  • For GitHub: the secret is only shown once — regenerate if lost
  • For Microsoft: make sure you copied the secret Value, not the Secret ID

Apple OAuth fails in development

  • Apple requires HTTPS for redirect URIs, even locally
  • Use a tunneling service (ngrok, Cloudflare Tunnel) or configure local TLS

OAuth works but user isn't created

  • Check logs: 
    RUST_LOG=oauth_signer=debug cargo run
  • The OAuth provider may not be returning an email address
  • Verify the OAuth scopes include email and profile

Relay Connection Issues

Bunker can't connect to relays

  • Check NOSTR_RELAYS in your .env — URLs must start with wss://
  • Verify network connectivity: 
    curl -I https://relay.damus.io
  • Check firewall rules allow outbound WebSocket connections on port 443

Clients can't find the bunker

  • Verify the bunker pubkey is correct in the client's connection string
  • Ensure client and bunker are using at least one common relay
  • Check that the bunker is running and connected: look for relay connection logs

Connection established but signing fails

  • Check if the user's identity assignment has expired
  • Verify the identity still exists in the database
  • Check logs for specific signing errors

Database Issues

"unable to open database file"

  • Check that the DATABASE_URL path is writable
  • Ensure the parent directory exists
  • Check file permissions

Database appears corrupted

  • SQLite uses WAL (Write-Ahead Logging) journal mode
  • Look for .db-wal and .db-shm files alongside the main database
  • Do not delete these files while the server is running
  • If corrupted, try: 
    sqlite3 oauth-signer.db "PRAGMA integrity_check;"

Assignment Issues

User can't see any identities in the picker

  • Check that active (non-expired) assignments exist for the user
  • Verify in admin panel: user ID matches and expiry is in the future
  • The background cleanup runs every 5 minutes — an assignment may have just expired

Signing fails with "assignment expired"

  • The identity assignment has expired since the connection was established
  • Create a new assignment in the admin panel with a longer duration
  • The user may need to reconnect after assignment renewal

Debugging Tips

Enable verbose logging:

RUST_LOG=oauth_signer=debug cargo run

Very verbose (includes HTTP requests):

RUST_LOG=oauth_signer=trace,tower_http=debug cargo run

Logs are structured JSON — pipe through jq for readability:

RUST_LOG=info cargo run 2>&1 | jq .

Test OAuth flow manually:

  • Visit http://localhost:3000/auth/google (or github/microsoft/apple) directly
  • Check that the redirect goes to the right OAuth provider
  • After callback, check logs for user creation

Check database state:

sqlite3 oauth-signer.db "SELECT id, email, oauth_provider FROM users;"
sqlite3 oauth-signer.db "SELECT id, label, pubkey FROM identities;"
sqlite3 oauth-signer.db "SELECT a.id, u.email, i.label, a.expires_at FROM user_identity_assignments a JOIN users u ON a.user_id = u.id JOIN identities i ON a.identity_id = i.id;"
